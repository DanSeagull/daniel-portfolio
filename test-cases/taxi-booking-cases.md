# Тест-кейсы: Вызов такси (базовый сценарий)

## Предусловия для всех тест-кейсов:
- Приложение Яндекс Go установлено и открыто
- Пользователь авторизован
- GPS включен
- Есть интернет

---

### TC-001: Успешный вызов такси "сейчас"

| Поле | Значение |
|------|----------|
| **ID** | TC-001 |
| **Название** | Вызов такси по адресу "Откуда" и "Куда" |
| **Приоритет** | Critical |
| **Предусловие** | На счету достаточно средств |
| **Шаги** | 1. В поле "Откуда" ввести "Москва, Тверская 1" <br> 2. В поле "Куда" ввести "Москва, Кремль" <br> 3. Нажать кнопку "Заказать" |
| **Ожидаемый результат** | 1. На карте отображается маршрут <br> 2. Появляется информация о водителе (имя, машина, рейтинг) <br> 3. Списывается предварительная сумма (холдирование) |
| **Пост-условие** | Поездка активна, статус "Ищем машину" |

---

### TC-002: Негативный сценарий — пустое поле "Куда"

| Поле | Значение |
|------|----------|
| **ID** | TC-002 |
| **Название** | Попытка заказа без указания точки назначения |
| **Приоритет** | High |
| **Шаги** | 1. В поле "Откуда" ввести адрес <br> 2. Поле "Куда" оставить пустым <br> 3. Нажать "Заказать" |
| **Ожидаемый результат** | Появляется ошибка: "Укажите точку назначения". Кнопка "Заказать" неактивна или выдает предупреждение. |

---

### TC-003: Изменение тарифа во время поиска

| Поле | Значение |
|------|----------|
| **ID** | TC-003 |
| **Название** | Смена тарифа с "Эконом" на "Комфорт" после начала поиска |
| **Приоритет** | Medium |
| **Шаги** | 1. Начать поиск такси с тарифом "Эконом" <br> 2. В процессе поиска (статус "Ищем") нажать на тариф и выбрать "Комфорт" |
| **Ожидаемый результат** | Поиск перезапускается с новым тарифом. Цена пересчитывается. Предыдущее холдирование отменяется. |
